Componentes

Cómo usar esta sección

Documentación breve de los componentes reutilizables del proyecto: selector, propósito, propiedades principales, variantes y ejemplos listos para copiar.

app-button

Propósito: Botón reutilizable con variantes visuales y soporte por teclado.

  • Selector: app-button
  • Inputs: variant ('primary'|'secondary'|'ghost'|'danger'), size ('sm'|'md'|'lg'), disabled, type, fullWidth
  • Output: (clicked) — EventEmitter
  • Estados: normal, deshabilitado
Guardar Cancelar 
<app-button variant="primary" size="md" (clicked)="onSave()">Guardar</app-button>

app-card

Propósito: Tarjeta de contenido con imagen, título, descripción y slot para acciones.

  • Selector: app-card
  • Inputs: title, description, image, variant ('basic'|'horizontal')
<app-card title="Pizza" description="Deliciosa" image="https://..."></app-card>

app-alert

Propósito: Mostrar mensajes de feedback (success, error, warning, info).

  • Selector: app-alert
  • Inputs: type ('success'|'error'|'warning'|'info'), closable
  • Output: (closed)
Guardado correctamente Error crítico 
<app-alert type="success" (closed)="onAlertClosed()">Operación OK</app-alert>

Inputs de formulario

Incluye app-form-input, app-form-select y app-form-textarea.

  • app-form-input — selector app-form-input. Inputs: label,type,name,placeholder,required,helpText,error,value. Emite valueChange.
  • app-form-select — selector app-form-select. Inputs: label, options (array {value,label}).
  • app-form-textarea — selector app-form-textarea. Inputs: label, placeholder, required.
<app-form-input label="Email" type="email" name="email" (valueChange)="onEmailChange($event)"></app-form-input>

Otros componentes

Componentes auxiliares y de layout: app-theme-switcher, app-calendar, app-dom-demo, app-register-form, app-login-form, app-header, app-footer.

  • app-theme-switcher — controla tema; emite themeChange.
  • app-calendar — calendario semanal con selección de días.
  • app-dom-demo — demo para manipulación DOM (Renderer2).
  • app-register-form / app-login-form — formularios compuestos listos para usar en páginas de Auth.
  • app-header / app-footer — layout principal (logo, navegación, tema).
<app-theme-switcher (themeChange)="onThemeChange($event)"></app-theme-switcher>

BEM (Block Element Modifier)

¿Qué es BEM?

BEM es una convención de nomenclatura para clases CSS que facilita la creación de interfaces de usuario escalables y mantenibles.

Principios básicos

  • Block: Representa un componente independiente y reutilizable.
  • Element: Parte de un bloque con significado propio.
  • Modifier: Variación de un bloque o elemento que altera su apariencia o comportamiento.

Ejemplos prácticos

Clases CSS siguiendo la convención BEM:


.boton {}

.boton--primario {}

.boton__texto {}

.boton--grande {}

.card {}

.card__titulo {}

.card__descripcion {}

.card--destacada {}
